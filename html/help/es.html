Ayuda

HeadingsMap es una extensión del navegador que muestra la estructura de encabezados de las páginas web. Además, puede identificar errores potenciales (como niveles de encabezado omitidos), ayudando al realizar revisiones de accesibilidad controles de calidad, auditorías SEO,...

La extensión también incluye la función opcional de mostrar la estructura de landmarks. Ambos resultados, los encabezados y los landmarks se muestran en un árbol que relaciona los niveles de anidamiento con la jerarquía del documento.

Características

  • Muestra la lista de encabezados con información sobre su nivel y estructura jerárquica, e identifica posibles errores
  • Si hay elementos que utilizan el atributo role con valor heading, también son listados. Su nivel es definido por el valor del atributo aria-level, y por defecto es 2 cuando no se utiliza
  • Cuando la página incluye más de un documento, extrae la información de cada uno de ellos
  • Al pulsar en los encabezados, la web hace scroll hasta la posición del encabezado/sección y lo resalta
  • Detecta los cambios en el DOM y cuando terminan actualiza los árboles
  • Tema oscuro (opcional)
  • Opcionalmente muestra los encabezados y secciones ocultas (ocultos para las tecnologías de asistencia). Cuando se muestran, el árbol incluye un icono para identificar cuáles fueron detectados como visibles y cuáles ocultos
  • Si el encabezado (o sección) es un ancla, el enlace en los resultados tiene el href apuntando a ella (útil para ser copiado directamente desde el árbol con la opción del menú contextual del navegador)
  • Si el contenido del encabezado/sección utiliza aria-label, se puede mostrar su valor (configurable en la sección de ajustes)
  • Disponible en múltiples navegadores. Utiliza la API "browserExtensions" por lo que es compatible con Chrome, Firefox, Microsoft Edge,...

Análisis de encabezados - configuración

Error si no hay h1 inicial
Es comúnmente aceptado que el primer encabezado del documento debe ser un h1. Con esta opción activa, e "identificar errores" también, se identifica como error si no se cumple la regla
Mostrar nivel de encabezado
Muestra el nivel de encabezado (1 a 6) que se utiliza como etiqueta (h1 a h6)
Considerar encabezados ocultos
Cuando el documento incluye encabezados que están ocultos para cualquier herramienta (no sólo visualmente ocultos, sino también ocultos para las tecnologías de asistencia), no se muestran en el árbol. Esta opción hace que se muestren en el panel como si fuesen visibles
Identificar anclas
Cuando los encabezados incluyen anclas, éstas muestran un icono en el árbol. Al hacer clic en el icono se copia la dirección URL dirección al portapapeles (igual que la acción "copiar dirección de enlace" del navegador)

Análisis de landmarks - configuración

Landmarks análisis activo
Activa y desactiva el análisis. Cuando está activo, ambos análisis se muestran como pestañas.
Error si banner, complementary, contentinfo o main no son top-level landmarks
Estos landmarks deben ser top-level landmarks.
Error si hay más de un banner, contentinfo, o main
Cada página puede tener como máximo un banner, contentinfo, o main.
Error si hay multiples instancias del mismo landmark sin label, o con el mismo label
Cuando una página tiene el mismo landmark repetido, deben tener un nombre y ser diferentes.

HTML 5 outline - configuración

Según el W3C, no hay navegadores ni tecnologías de asistencia que lo soporten, por lo que tener un buen resultado en el test apenas repercute en la accesibilidad final. Según el W3C, cualquier implementación y uso será rastreada, y puede dar lugar a cambios significativos en el futuro del algoritmo.

Mostrar índice de sección
Según el algoritmo, cada sección tendría un índice en el árbol. Este ajuste muestra/oculta dicho valor
Mostrar etiqueta de sección
Con este ajuste, el árbol muestra la etiqueta HTML que genera las secciones.
Considerar secciones ocultas
Cuando el documento incluye secciones ocultas (no sólo visualmente ocultas, sino también ocultas para las tecnologías de asistencia), no se muestran en el árbol. Al activar esta opción, se muestran. En documentos largos, esto podría afectar al rendimiento ya que requiere comprobar todos los elementos del DOM
