Aide

HeadingsMap est une extension de navigateur qui affiche la structure des titres des pages web. De plus, elle peut identifier les erreurs potentielles (telles que les niveaux de titres manquants), ce qui facilite la réalisation d’audit d’accessibilité, de contrôles qualité, d'audits SEO, ….

Fonctionnalités

  • Affiche une liste de titres avec des informations sur leur niveau et les ruptures dans la hiérarchie de titres.
  • Si des éléments utilisent l’attribut role heading, ils sont également listés. Leur niveau est basé sur la valeur de l'attribut "aria-level" (si non défini, la valeur par défaut est 2).
  • Liste des sections avec des informations sur les titres et des avertissements pour les erreurs de structure.
  • Obtient les informations à partir du document principal ainsi que des documents à l'intérieur des iframes.
  • En cliquant sur les titres, le document défile jusqu'à la position du titre ou de la section et l’entoure.
  • Détecte les changements dans le DOM et, une fois terminés, met à jour les arbres.
  • Thème sombre (option)
  • Optionnellement, il affiche les titres et les sections masqués (masqués pour les technologies d'assistance). Lorsqu'ils sont affichés, une icône est ajoutée aux éléments de l'arborescence pour identifier ceux qui ont été détectés comme étant visibles ou masqués.
  • Si le titre (ou la section) est une ancre, le lien dans les résultats possède l'attribut "href" qui pointe vers celui-ci (utile pour être copié directement à partir de l'arborescence avec l'option du menu contextuel du navigateur).
  • Si le contenu du titre/section utilise "aria-label", sa valeur peut être affichée (propriété facultative configurable dans la section des paramètres).
  • Version pour plusieurs navigateurs. Elle utilise l'API "browserExtensions", ce qui la rend compatible avec Chrome, Firefox, Microsoft Edge, …

Arborescence de HeadingsMap - Paramètres

Erreur si pas de h1 initial
Il est communément accepté que le premier titre du document doit être un h1. Avec cette option activée, ainsi que "identifier les erreurs", le non-respect de cette règle est considéré comme une erreur.
Niveau de titre
Affiche le niveau du titre (de 1 à 6) qui est utilisé comme balise (h1 à h6).
Afficher les titres masqués
Lorsque le document comprend des titres masqués pour n'importe quel outil (non seulement visuellement masqués, mais également masqués pour les technologies d'assistance), ils ne sont pas affichés dans l'arborescence. Lors de l'activation de cette option, ils sont également affichés.
Afficher les ancres
Lorsque les titres incluent des ancres, ils affichent une icône dans l'arborescence. En cliquant sur l'icône, l'adresse URL est copiée dans le presse-papiers (comme l'action "copier l'adresse du lien" du navigateur).

Landmarks analysis active
It activates or deactivates the landmarks analysis. When the analysis is active, both headings and landmarks analysis appear as tabs.
Error if banner, complementary, contentinfo or main are not top level landmarks
These landmarks should be top-level landmarks.
Error if more than one banner, contentinfo, or main
Each page may have at most one of each type.
Error if repeated landmarks without label, or with same label
When there are more than one landmark with same role, they should include a label to differentiate them.

HTML 5 outline - Paramètres

Selon le W3C, aucun navigateur ou technologie d'assistance ne le prend en charge, donc obtenir un bon résultat dans le test a à peine un impact sur l'accessibilité finale. Selon le W3C, toute mise en œuvre et utilisation de ces différentes propositions sera suivie et pourra entraîner des changements importants dans l'algorithme de la structure.

Afficher l'index de la section
Selon l'algorithme, chaque section aurait un index dans l'arborescence. Ce paramètre affiche/masque cette valeur.
Afficher la balise section
Avec ce paramètre, l'arborescence affiche la balise HTML qui génère les sections.
Afficher les sections masquées
Lorsque le document inclut des sections masquées (non seulement visuellement masquées, mais également masquées pour les technologies d'assistance), elles ne sont pas affichées dans l'arborescence. Lors de l'activation de cette option, elles sont affichées. Dans les documents longs, cela pourrait affecter les performances car cela nécessite de vérifier tous les éléments du DOM.
