ヘルプ

HeadingsMap は Web ページの見出し構造を表示するブラウザ拡張機能です。 さらに、潜在的なエラー (ヘッダーレベルの欠落など) を特定できるため、アクセシビリティチェック、品質管理、SEO 監査などを実行するときに役立ちます。

特徴

  • 見出しのリストとそのレベルに関する情報、および見出しが階層構造を壊しているかどうかを表示します。
  • もし role 属性の値が heading である要素があれば、それらもリストに表示されます。 それらのレベルは、aria-level 属性の値に基づきます (定義されていない場合、デフォルトは 2)。
  • ヘッダー情報と構造上のエラーに関する警告があるセクションのリストです。
  • メインドキュメントと iframe 内のドキュメントから情報を取得します。
  • ヘッダーをクリックすると、文書はそのヘッダー/セクションの位置までスクロールし、強調表示されます。
  • DOM の変更を検出し、変更が完了するとツリーを更新します。
  • ダークテーマ (オプション)
  • オプションで、非表示のヘッダーとセクション (支援技術のために非表示) が表示されます。 それらが表示されると、ツリー内の項目にアイコンが追加され、拡張機能によってどの項目が表示または非表示として検出されたかを識別します。
  • ヘッダー (またはセクション) がアンカーの場合、結果内のリンクにはそれを指す href が含まれます (ブラウザーのコンテキストメニューのオプションを使用してツリーから直接コピーする場合に便利です)。
  • ヘッダー/セクションのコンテンツで aria-label が使用されている場合、その値を表示できます (オプションのプロパティは設定セクションで構成可能)
  • 複数のブラウザに対応したバージョンです。 "browserExtensions API" を使用しているため、Chrome、Firefox、Microsoft Edge などと互換性があります。

HeadingsMap ツリー - 設定

h1 タグが初めにない場合エラー
ドキュメントの最初のヘッダーは h1 でなければならないことが一般的に受け入れられています。 このオプションがアクティブで、"エラーの識別" も有効な場合、このルールの欠如はエラーとみなされます。
見出しレベル
タグ (h1 から h6) として使用される見出しレベル (1 から 6) が含まれます。
非表示の見出しの考慮
ドキュメントにツールに対して非表示になっているヘッダー (視覚的に非表示になっているだけでなく、支援テクノロジに対しても非表示になっているヘッダー) が含まれている場合、それらはツリーには表示されません。 このオプションを有効にすると、それらも表示されます。
アンカーの特定
ヘッダーにアンカーが含まれている場合、ツリーにアイコンが表示されます。 アイコンをクリックすると、URL アドレスがクリップボードにコピーされます (ブラウザの "リンク アドレスをコピー" アクションと同じ)

HTML 5 outline - 設定

W3C によると、これをサポートするブラウザや支援技術は存在しないため、テストで良い結果を得ても、最終的なアクセシビリティにはほとんど影響しません。 W3C によると、これらのさまざまな提案の実装と使用状況は追跡され、将来的にアウトラインアルゴリズムに大幅な変更が生じる可能性があります。

セクションインデックスを表示
アルゴリズムによれば、各セクションはツリー内にインデックスを持ちます。 この設定はその値を表示/非表示にします。
セクションタグを表示
この設定では、セクションを生成する HTML タグがツリーに表示されます。
非表示のセクションの考慮
ドキュメントに非表示のセクション (視覚的に隠されているだけでなく、支援技術のためにも隠されている) が含まれている場合、それらはツリーには表示されません。 このオプションを有効にすると、それらが表示されます。 長いドキュメントでは、DOM 内のすべての要素をチェックする必要があるため、パフォーマンスに影響する可能性があります。
